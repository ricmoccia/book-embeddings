Detailed Description

Enumerator of all valid orderings of the children of cutpoints in a rooted BCT.

Given a BCT rooted at a block, every cutpoint c has a (possibly empty) list of source-type (type-0) and sink-type (type-2) block children. The children inside each of those two lists can be permuted freely; this class iterates over the Cartesian product of all such permutations.

The current state of the iteration is kept in permutations, indexed by cutpoint value and edge type. After each call to getNextPermutation permutations is advanced (lexicographically) for the next call.

Because the advance happens DURING getNextPermutation, the state of permutations at return-time no longer matches the order that was just produced. To allow callers (e.g. BCT/FPQ printers) to know the permutation actually used for the order they are about to render, we expose a separate snapshot lastUsedPermutations that is filled with the state just before the advance.

std::vector< int > BCTPermutationEnumerator::getNextPermutation ( )
inline

Compute the next valid ordering of the biconnected components following the current cutpoint permutations, then advance the state for the next call.

On entry, permutations holds the next permutation to be applied. We first snapshot it into lastUsedPermutations, then use it to build the component order, and finally advance permutations with std::next_permutation. This way external code can render the BCT/FPQ with children in the exact order that produced the returned layout.

Returns
The order in which the biconnected components must be visited to produce the corresponding 1-stack layout.
