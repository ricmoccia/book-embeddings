Detailed Description

A node of the FPQ-tree.

Each node has a unique id within the tree. Depending on type either blockIndex (for F_NODE_BLOCK) or value (for LEAF) is meaningful. The order of children is significant for F-nodes and irrelevant for P-nodes (but it is kept stable for rendering).
